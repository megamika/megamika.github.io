Level scripting

Problem
The game is in the metroidvania genre so that required us to have some areas that could only be acessed after a certain condition has been met or a certain location was reached.

Solution
There is a system in the game which allows a level designer to pair an event in the game to be connected to another event without the need to write an extra script.

- Opening a door after an enemy has been killed
- Opening a latch after the lever has been pulled

Ability unlocks

The game features several upgrades to give the player a feeling of progression and to change up the gameplay.

Sword

Air dashes

First iteam you recieve in the game is just your sword. The goal was to have as little of a tutorial in the game as possible so the way the player would be taught how to play is by restricting his abilities untill he has proven that he has understood how to use his other abilities so the sword will only be granted to the player after he has done some basic platforming.

This ability allows you to dash up to the areas you could not reach before. It was added as one of the ways we restrict some areas from being visited before the player has unlocked this ability. There are a few things that could have been improved about it however.

- It's a pretty linear way to interact with the game, you only get to be thrown into one direction thus limiting your options.

- There was no clear way to predict how far the thrusters would throw the player so a lot of the time they just overshoot or undershoot.

Dash and a ghost dash

After exploring the world for some more time you get the ability to dash. That'll give you some extra movility.

Another upgrade will make it so your dash will allow you to phase through lazers and projectiles. That does not only serve as a way for the player to explore more areas, but also allows the player to have more of an advantage when dealing with projectiles.

Map

By pressing TAB you can open up the map showing you the whole area and also the trail of where you have just explored. Pretty neat when you are stuck and not sure where to go next.

The system was added at the very last week of development cause of the realization that the map of the game would be too big to mentally remember.

Dynamic obstacles

To offer variety to the levels any obstacle could be made to be dynamic by adding some simple components to them. Any obstacle could be
- Moved back and forth
- Rotated on a periodic basis
- Be toggled at an interval
That added variety without the need to add more prefabs.

Emergent enemy behaviour

Enemie AI itself is very basic, they basically do one or 2 things on a loop, but together they often lead to unexpected behaviours.

There is no real "team" system in the game so the player character and the enemy characters all interact in the same way with each other. I have planned to add a check to make sure the enemies cannot damage each other, but the way the enemies would hurt each other was very satisfying so it was kept that way.
