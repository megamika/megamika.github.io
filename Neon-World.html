The repository

Player movement

Air thrust

The air thrust was the main feature of the game and where most of the fun was derived from. I have came up with it during a brainstorming session and the idea of that kind of a dash was derived from my other game Space Juice.

The movement in the game was initially supposed to be physics based, but ended up being more of a rigid "hand crafted" movement with hints of fake physics.

Character unlocks

Problem
It was important for the player to have some kind of a motivation to keep playing and also to have a choice of picking the look they liked more. 

Solution
The solution to this would be different characters you could unlock by finishing cerain levels throughout the game. 

Failures
- The descriptions were pretty vague and it was unclear what to do to unlock a specific skin
- There were no notification of when you would unlock the character so all you could do is just frantically check if you have actually got the character you wanted

Destructable iteams

Problem
There were a few explicit rewards in the game such as improving your time in the levels or unlocking cosmetic skins, however there wasn't much in terms of implicit rewards.

Solution
The game features a simple, but fairly effective and satisfying system for making destructable iteams. 

Camera directing

Problem
Sometimes the levels required more platforms to be seen at the same time and the player would not be able to make some jumps without a leap of faith.

Solution
A system which would pan the camera out to showcase a predefined location and size would do the trick.

Level creation / management

One of the goals for this game was to have a variety of levels to keep the player occupied for a longer amount of time, so to see that vision play out there is a system in the game which allows the level designer to quickly add levels and have UI adjust automatically.

Level designer only has to worry about arranging a basic node system to define which levels are in which sub category and the code will adjust the UI accordingly.

The algorithm will also make sure that the level before is completed before the new level is allowed to be played as well as only show the level category when at least one level from that category is allowed to be played.
